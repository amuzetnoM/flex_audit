Overview

Flex Audit is a comprehensive, extensible auditing framework for code and configuration. It combines regex scanning, AST analysis for Python, vulnerability heuristics, and policy-driven reporting. This documentation summarizes what Flex Audit can do and how to operate it in CI or local environments.

    Key Features

    Secret Detection

    Detects API keys, passwords, credentials, and private keys across many languages.

    Static Vulnerabilities

    SQLi, XSS, command injection, unsafe deserialization patterns identified via regex and language-aware checks.

    Code Quality & AST

    Python AST-based checks catch mutable defaults, bare exceptions, and questionable patterns.

    CI Integration

    Exports SARIF for GitHub Actions and provides helpful CLI flags for CI policy management.

    Quick Usage

    python -m flex_audit.src /path/to/repo --format all --skip-third-party

    Use --skip-third-party to ignore virtualenv and vendor folders like venv or site-packages.

    Report Outputs

    Flex Audit supports Markdown, HTML, JSON, and SARIF outputs. SARIF is the recommended export for machine processing and CI integration.
