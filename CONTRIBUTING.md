# Contributing to Flex Audit

Thanks for considering contributing to Flex Audit — this doc covers the basics to help you get started.

## Reporting Issues
- Open issues for bugs, unexpected behavior, or missing features.
- Include a reproducible example and platform environment information (OS, Python version).
- If the issue is a security vulnerability, please follow the guidelines in `SECURITY.md`.

## Pull Requests
- Fork the repo and make changes on a feature branch.
- Run linters and tests locally before opening a PR.
- Keep PRs focused and include screenshots or outputs when relevant.
- Please include any new tests for new features or bug fixes.

## Coding Standards
- Use `black` for formatting and `ruff`/`flake8` for linting.
- Type hints are encouraged and helpful for maintenance.
- Follow the project’s existing code structure and patterns.

## Compatibility
- This repository targets Python 3.12+ (see README).
